# Nova Architect ‚Äî Multi-Stack Projects

## Overview

Nova supports projects with separate frontend and backend services. When you scaffold a multi-stack project (e.g. "Next.js + C# .NET"), Nova:

1. Creates frontend in the project root (`.`)
2. Creates backend in `backend/` subdirectory
3. Automatically sets `frontend` and `backends` in `nova.toml`
4. Indexes both services separately
5. Routes AI-generated files to the correct directory

## How Nova Uses frontend/backends

### Indexing

When `frontend` and `backends` are set, Nova **only scans those directories** instead of the entire project. This:
- Speeds up indexing
- Prevents confusion between frontend and backend code
- Provides clearer context to the AI

### AI Context

The compressed project context sent to the AI includes:
```
Service architecture:
  Frontend: .
  Backends: backend
```

This tells the AI where to create files:
- React components ‚Üí `components/`
- Pages ‚Üí `app/`
- API endpoints ‚Üí `backend/Controllers/` (for .NET)

### Task Routing

When you say "add a login form", Nova knows to create:
- `components/LoginForm.tsx` (not `backend/components/LoginForm.tsx`)
- `app/login/page.tsx` (not `backend/app/login/page.tsx`)

When you say "add a users API endpoint", Nova creates:
- `backend/Controllers/UsersController.cs` (for .NET)
- Or `app/api/users/route.ts` (for Next.js API routes)

## Manual Configuration

For existing multi-service projects, set the directories manually:

```toml
# nova.toml
[project]
frontend = "packages/web"
backends = ["packages/api", "packages/worker"]
devCommand = "npm run dev"
port = 3000
```

## Entity Management

For advanced multi-service setup, use the manifest:

```bash
nova entity add
```

This lets you define:
- Services with roles (frontend, backend, worker, gateway)
- Databases with connection details
- Boundaries (writable, readonly, ignored paths)

```toml
# .nova/manifest.toml
[[services]]
name = "web"
role = "frontend"
path = "."
framework = "next.js"

[[services]]
name = "api"
role = "backend"
path = "backend"
framework = "dotnet"

[[databases]]
name = "postgres"
engine = "postgresql"
connectionEnvVar = "DATABASE_URL"

[boundaries]
writable = ["app", "components", "lib", "backend/Controllers"]
readonly = ["backend/Migrations"]
```

## Tips

### Running Both Services

Nova only starts the **frontend** dev server. Start the backend separately:

```bash
# Terminal 1: Nova handles frontend
nova start

# Terminal 2: Start backend manually
cd backend && dotnet run
```

### Cross-Service Communication

When the AI creates code that calls the backend API, it uses environment variables:

```typescript
// Generated by Nova
const res = await fetch(`${process.env.API_URL}/api/users`);
```

Nova's SecretConsole will prompt you to enter `API_URL` (e.g. `http://localhost:5000`).

### Shared Types

For TypeScript projects with shared types between frontend and backend, create a `shared/` directory and add it to both `frontend` and `backends`:

```toml
[project]
frontend = "."
backends = ["backend"]
```

The AI will see types from both services and maintain consistency.
